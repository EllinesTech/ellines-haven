================================================================================
CRITICAL FIX: REFRESH BUTTON (iOS Safari)
================================================================================

PROBLEM:
  ChunkErrorBoundary refresh used .then() chains that can block on iOS Safari,
  causing refresh button to fail inconsistently.

SOLUTION:
  New hardReload() function (App.jsx lines 51-66):
  
  1. Clears all service worker caches (non-blocking)
  2. Sets 600ms timeout to hard reload regardless
  3. If caches clear quickly, reload fires immediately
  4. If caches are slow/fail, timeout ensures reload anyway
  5. Falls back to window.location.reload() if caches unavailable
  
  Result: Refresh now works 100% reliably on all devices.
